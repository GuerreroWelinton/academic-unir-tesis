# Welcome to the ZGames UI Library

A modern, accessible, and reusable UI component library for iGaming experiences.

## What is ZGames UI?

ZGames UI is a design system and component library built with **Angular 21** that provides:

- Reusable standalone components
- WCAG-oriented component development
- Token-based theming
- Storybook-driven documentation
- A container-presentational architecture

## Why ZGames UI?

This library exists to:

1. Reduce technical debt by centralizing reusable UI
2. Enforce visual consistency across products
3. Accelerate delivery with documented components
4. Improve accessibility quality and evidence

## Getting Started

### Installation

```bash
npm install @zgames/ui
```

### Importing Components

All components are **standalone** and can be imported directly:

```typescript
import { Component } from '@angular/core';
import { ZgButtonComponent } from '@zgames/ui';

@Component({
  selector: 'app-my-component',
  standalone: true,
  imports: [ZgButtonComponent],
  template: ` <zg-button variant="primary" (clicked)="handleClick()"> Click me </zg-button> `,
})
export class MyComponent {
  handleClick() {
    console.log('Button clicked!');
  }
}
```

### Explore Components

Use the Storybook sidebar to:

- view live component examples
- test states and variants
- review accessibility behavior
- copy usage snippets

## Accessibility

Components are validated against **WCAG 2.1 Level AA**.

By default, Storybook's **a11y** addon runs WCAG 2.1 AA checks.

Use each story's **Accessibility** tab to review:

- issues to fix
- passed checks
- additional manual checks

## Tech Stack

- **Angular 21**: standalone components with Signals
- **Storybook 10**: component explorer and docs
- **axe-core**: automated accessibility checks
- **Chromatic**: visual regression checks
- **Vitest**: unit testing
- **TypeScript**: strict typing

## Usage Boundaries

Use ZGames UI for:

- reusable presentational UI components
- token-based, documented interface building

Avoid using ZGames UI for:

- business/domain logic
- API/data orchestration or app-level side effects
